DL Image Caption Generator

DL-Image-Caption-Generator-

CNN-LSTM encoder-decoder that generates natural language descriptions for images, trained on Flickr8k.

Python Deep Learning CNN LSTM Transfer Learning NLP Computer Vision

Overview

A deep learning pipeline that automatically generates natural language descriptions for images. The system uses a CNN-LSTM encoder-decoder architecture to learn the mapping between visual features and textual descriptions, trained on the Flickr8k dataset (~8,000 images, 5 captions each).

Model Architecture

Merge architecture with two independent branches fused at the decoding step:

Image → ResNet50 (pretrained) → Dense(256) —\
                                                 → add → Dense(256) → softmax(vocab)
Caption → Embedding(256) → LSTM(256) —/
  • Encoder: ResNet50 with ImageNet weights — extracts 2048-d visual features via transfer learning
  • Decoder: Embedding + LSTM (256 units) — models sequential language structure with gating for long-range dependencies
  • Merge strategy: element-wise addition keeps vision and language independent until final decoding, producing more stable training

Training Strategy

  • Loss: categorical cross-entropy — multi-class word prediction at each timestep
  • Optimizer: Adam — adaptive learning rates for fast convergence
  • Regularization: Dropout (0.5) on image features and embeddings to prevent overfitting on the small dataset
  • Early stopping: patience-based with best-weight restoration
  • Data feeding: on-the-fly generator with teacher forcing to avoid loading the full expanded dataset into memory

Key Design Decisions

  • ResNet50 over VGG16 — deeper yet faster due to bottleneck blocks, with residual connections solving vanishing gradients
  • LSTM over vanilla RNN — gating mechanisms capture dependencies across multi-word captions where vanilla RNNs fail
  • Merge over inject — keeps modalities independent until fusion, shown to produce comparable or better results on small datasets
  • Transfer learning — training a CNN from scratch on ~8k images would massively overfit; ImageNet features already encode edges, textures, and objects

Results

  • Training converges within ~6–11 epochs depending on early stopping patience
  • Model correctly identifies objects, people, and colors in many test images
  • Greedy decoding generates captions word-by-word from startseq to endseq
  • Limitations on complex scenes reflect the small dataset size — attention mechanisms or beam search could improve accuracy
