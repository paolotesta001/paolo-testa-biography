Overview

A multi-agent AI-powered nutrition tracking system built with FastAPI and Gemini 2.0-Flash. The platform uses 5 specialized agents to help users log meals, plan nutrition, and receive personalized health advice — all through natural language conversation.

The system supports dual deployment modes (monolith and microservices), three interchangeable communication protocols, full Kubernetes orchestration with horizontal auto-scaling, and ML-based food image classification using MobileNetV2 trained on Food-101.

Architecture

Hub-and-spoke multi-agent architecture. An Orchestrator receives user messages, classifies intent via LLM, and routes requests to the appropriate specialized agent. Agents communicate through pluggable protocols and share a common PostgreSQL database via a dedicated DB Writer agent.

  • Orchestrator — Intent classifier & router for all user messages
  • Food Logger — Parses food descriptions into structured nutrition data + ML image classification
  • Meal Planner — Generates personalized daily/weekly meal plans with shopping lists
  • Health Advisor — Analyzes nutrition history and provides evidence-based advice
  • DB Writer — Handles all database operations (no LLM dependency)

Communication Protocols

Three interchangeable protocols, selectable at runtime:

  • A2A (Agent-to-Agent) — Full metadata, 36-char UUID, JSON serialization
  • PNP (Paolo Nutrition Protocol) — Custom-designed, ~2x lower latency than A2A, optional MessagePack binary serialization
  • TOON — Fastest on containers (72ms avg, tightest P99 at 96ms)

ML Food Classifier

  • MobileNetV2 with transfer learning on Food-101 (101 categories, 101K images)
  • 2-phase training: classifier head (5 epochs) + full fine-tune (10 epochs)
  • 82.28% test accuracy, ~5ms inference on T4 GPU
  • TorchScript format: 1.6x faster inference, 2.3x less RAM than standard .pth

Deployment & Infrastructure

  • Dual deployment: monolith (single process) or microservices (5 containers) — switchable via one env variable
  • Kubernetes with HPA auto-scaling (1-3 replicas per agent at 70% CPU)
  • Nginx reverse proxy with rate limiting, gzip, and 120s LLM timeout
  • Automated daily PostgreSQL backups with 7-day retention
  • Full execution tracing with trace ID propagation across all agent hops
  • LLM reliability layer: retry with feedback, rule-based fallback, per-user token budgets

Key Results

  • Containerization improves all protocols by 10-24x vs monolith
  • TOON wins on containers: 72ms avg latency, 13.82 msg/s throughput
  • PNP achieves 2x lower latency than A2A through compact IDs and optional binary serialization
  • ML model: 82% accuracy, 0.22 img/s on CPU, ~5ms on GPU
